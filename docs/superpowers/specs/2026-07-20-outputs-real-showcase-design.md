# 「产出」区块真实展示:3D 跟手卡 + 真实成品图

日期:2026-07-20 · 状态:已获用户批准

## 目标

「产出」区(`#outputs`,139 计数器 + 六张纯文字卡)加入产品真实画面,
形态选定为「3D 跟手卡」:卡内嵌真实成品截图,悬停时卡片随指针轻微 3D
倾斜、截图反向视差、光斑扫过——与首屏 Hero 3D 卡片墙同一套视觉语言。

## 素材:6 张产品真实截图

| 卡片 | 去产品里截什么 |
|---|---|
| 做 PPT / 演示 | 画布的幻灯片成品页 |
| 做表格 | 表格类技能的运行结果 |
| 写方案 / 文档 | 方案写作模式界面(封面→目录→正文) |
| 生成图片 / 海报 | 画布的图片成品 |
| 生成视频 | 画布的视频面板 |
| 数据报告与图表 | 可交互的 HTML 报告画面 |

- 截图方法:CDP `Page.captureScreenshot` clip + scale:2(禁用
  `Emulation.setDeviceMetricsOverride`,会点错位/花屏)。
- 不虚构原则:某类没有现成成品时,用画布社区模板的真实模板顶上,
  或当场真跑一次生成;绝不做假图。
- 隐私处理:侧栏真实会话标题 DOM 替换为演示文字(仅显示层)。
- 红线:不按 Enter 发消息、不点技能开关、不点语言卡片、不乱改设置;
  改了产品设置要等 5 秒以上再退出。
- 产出:`public/screens/outputs/*.webp`(cwebp -q 82,与现有素材同规格)。

## 组件改造(components/sections/Outputs.tsx)

卡片从纯文字变「上图下文」:
- 上半:圆角小窗嵌真实截图(浅色产品窗口浮在深色卡上,与 Screens 区同语言)。
- 下半:保留现有图标、标题、说明、扩展名徽章。

## 动效(motion/react,不引新依赖)

- 3D 跟手倾斜:指针相对卡心位置 → rotateX/rotateY(约 ±5°),
  useSpring 平滑,移开回弹归零。
- 截图反向视差:卡倾斜时截图反向小幅位移 + 微放大。
- 光斑:跟随指针的径向渐变柔光层。
- 入场:保留 RevealGrid 错峰弹簧。
- 降级:prefers-reduced-motion → 全静态;触屏(无 hover)→ 静态截图,不倾斜。

## 数据层(lib/content.ts)

`outputCards` 每项加截图路径字段(及 alt 文案),中英双语结构不变。

## 验证

Chrome DevTools MCP 打开 localhost:3000/#outputs,确认动效手感、
入场、降级路径;截图留档。
